## 一、产品概述

### 1.1 当前产品

ScoreCheck 是一个**轻量级成绩查询系统**，核心场景：
- 学生输入「姓名 + 身份证号」查询个人成绩
- 教师登录后台管理成绩数据（导入、看板、用户管理）

### 1.2 产品现状

| 维度 | 当前状态 |
|------|---------|
| 用户角色 | 教师、学生（无注册机制） |
| 数据隔离 | 按 className 多租户隔离 |
| 认证方式 | Sa-Token + 滑块验证码 |
| 数据录入 | Excel 批量导入 |
| 成绩展示 | 学生端单条查询、教师端聚合看板 |

---

## 二、核心问题分析

### 2.1 当前系统的局限性

1. **功能单一** — 仅满足基本查询，缺乏数据分析能力
2. **用户体验断层** — 教师端和学生端的 UI/UX 风格割裂
3. **数据扩展性差** — 不支持多维度筛选、排名趋势分析
4. **无消息通知** — 成绩更新后学生不知道
5. **移动端适配不足** — 学生查询场景主要在手机

## 三、产品方向规划

### Phase 1：基础加固（P0 本期）

**目标：修复已知 bug，夯实基础体验**

| 优先级 | 模块 | 需求描述 | 验收标准 |
|-------|------|---------|---------|
| P0 | 前端进度条 | 根据科目满分动态计算进度百分比 | 150 分的科目进度条不超过容器宽度 |
| P0 | 登录流程 | 后端登录接口返回 className，前端存储 | 教师看板只显示本班数据 |
| P1 | 参数校验 | 身份证号正则校验、姓名为必填 | 前端拦截非法输入，后端返回明确错误 |
| P1 | DTO/VO 分离 | 后端返回 ScoreVO 不含 delFlag | 前端类型定义与实际响应一致 |
| P1 | 登录限流 | 连续 5 次失败锁定 15 分钟 | GlobalExceptionHandler 中实现 |
| P2 | Token 管理 | 前端增加 token 过期校验逻辑 | localStorage 存储 ttl，过期自动跳转登录 |
| P2 | 响应拦截器优化 | 网络错误统一友好提示 | 不显示 raw network error |
| P2 | 骨架屏 | 教师看板和数据导入加载态 | 替代空白等待页 |

### Phase 2：功能增强（中期规划）

**目标：丰富功能，提升实用价值**

#### 2.1 成绩分析模块

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 班级排名 | 按总分或单科排名 | P0 |
| 成绩趋势 | 同学生历次考试成绩折线图 | P1 |
| 班级对比 | 不同班级/不同考试横向对比 | P1 |
| 分数段分布 | 柱状图展示 60-、60-70、70-80、80-90、90+ 人数 | P2 |
| 薄弱科目识别 | 自动标出低于班级均分的科目 | P2 |

#### 2.2 数据管理模块

| 功能 | 说明 | 优先级 |
|------|------|--------|
| Excel 导出 | 按班级/考试导出成绩 | P0 |
| 成绩修正 | 教师手动修改单笔成绩，记录操作日志 | P1 |
| 考试管理 | 创建、归档、删除考试 | P1 |
| 批量删除成绩 | 按考试一键清空 | P2 |

#### 2.3 学生端增强

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 成绩推送通知 | 新成绩录入后站内信通知 | P1 |
| 历史成绩查询 | 学生查看所有历史考试 | P2 |
| 教师评语 | 教师可为每次考试写评语 | P2 |

### Phase 3：形态演进（长期愿景）

| 方向 | 说明 | 预估阶段 |
|------|------|---------|
| **移动端适配** | 学生端响应式重构，优先覆盖手机场景 | Phase 3.1 |
| **微信小程序** | 学生扫码查分，教师手机端管理 | Phase 3.2 |
| **开放 API** | 提供 RESTful API 供第三方集成 | Phase 3.3 |
| **SaaS 化** | 多学校独立租户，按学校收费 | Phase 4 |

## 四、技术架构演进

### 4.1 当前技术栈

```
Spring Boot 3.2.10 (Java 17)
├── Sa-Token 1.45.0      (认证)
├── MyBatis-Plus 3.5.7   (ORM + 多租户)
├── EasyExcel 3.3.2      (Excel 导入)
└── BCrypt               (密码加密)

Vue 3.5.12 + TypeScript
├── Element Plus 2.13.7  (UI 组件)
├── Pinia 3.0.4          (状态管理)
├── Vue Router 4.6.4     (路由)
└── Axios 1.15.2         (HTTP 客户端)
```

### 4.2 技术债务清单

| 债务项 | 影响 | 解决阶段 |
|--------|------|---------|
| DTO/VO 未分离 | 敏感字段泄露 | Phase 1 |
| 无单元测试 | 回归风险高 | Phase 2 |
| Excel 导入并发不安全 | 数据错乱 | Phase 1 |
| 接口参数无校验 | 脏数据入库 | Phase 1 |
| 无 API 文档 | 前后端协作效率低 | Phase 2 |
| 缺少 Docker Compose | 部署门槛高 | Phase 2 |
| 前端类型未共享 | 前后端类型不一致 | Phase 3 |

## 六、非功能性需求

| 类别 | 要求 |
|------|------|
| **性能** | 成绩列表查询 < 500ms，Excel 导入 1000 条 < 10s |
| **可用性** | 核心接口 SLA 99.9%，优雅停机 |
| **安全性** | 密码加密存储，敏感数据脱敏，HTTPS 强制 |
| **可维护性** | 单元测试覆盖率 > 60%，CI/CD 流水线 |
| **可扩展性** | 支持水平扩展，数据库读写分离预留 |
